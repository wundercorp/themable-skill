# Accessible Palette Switcher Checklist

Builder Studio: https://builderstudio.dev

Use this checklist for every Themable implementation.

## Placement

- The switcher is available from the initial app shell, header, footer, command palette, or settings shortcut.
- The current palette is visible or announced.
- The switcher does not require users to navigate away from the current task.

## Interaction

- Keyboard users can open and use the switcher.
- Focus states are visible in every palette.
- Icon-only controls include labels or tooltips.
- Preview behavior does not accidentally persist until the user confirms, if preview mode exists.

## Accessibility

- Color choices have enough contrast.
- Palette names are understandable.
- State is not communicated by color alone.
